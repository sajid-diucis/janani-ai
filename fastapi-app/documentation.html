Janani AI: The Complete Technical Blueprint

Janani is a high-availability, safety-critical maternal health OS. It is designed to operate in low-bandwidth, low-literacy environments while maintaining strict clinical compliance (WHO-standard).

1. Technical Stack & Architecture

Frontend Layer

  • Framework: Next.js 15 (CSR-Only for Static Portability).
  • Core Design: Glassmorphic UI using Vanilla CSS 3.0 variables; custom micro-animations for user engagement.
  • Offline Persistence: PWA with Service Workers (next-pwa) and IndexedDB (offline-manager.ts) for caching data bundles.

Backend Layer

  • Language: Python 3.12 (Asynchronous).
  • API Framework: FastAPI with Pydantic v2 data validation.
  • Security: SHA-256 Auth system; Production-hardened with pysqlite3-binary for ChromaDB compatibility on Linux.

Intelligence & Edge Computing

  • Core LLM: Gemini 2.0 Flash (Primary) / DeepSeek (Fallback).
  • Vision Engine: Groq Vision (meta-llama/llama-4-scout) for sub-second prescription and food safety analysis.
  • Local RAG: ChromaDB Vector Store.
  • Edge Embeddings: Custom SHA-384 hashing algorithm (768-dim) for high-speed, offline-capable search without external deps.

3. Data & Clinical Models

Maternal Risk Profile

Extremely high-density data object containing:

  • Obstetric History: Gravida (total pregnancies), Para (completed deliveries), LMP (Last Menstrual Period).
  • Automated Bio-Logic: Dynamic EDD (Expected Due Date) and Current Week calculation.
  • Risk Assessment: BMI calculation; automated flags for Anemia, Hypertension, and Gestational Diabetes.
  • Social Factors: Distance to hospital, income, and education level factored into risk scoring.

WHO Knowledge Graph

A directed graph of maternal health facts:

  • Entities: Symptoms, Nutrition, Exercises.
  • Relations: Maps symptoms to WHO-mandated clinical guidelines (e.g., "Blurred Vision" -> "Assess for Pre-eclampsia").
